If / Else and Conditional Operators


A fundamental feature of most programing languages is the ability to ask a simple question that gives a true or false answer, and in GML this is achieved using the if keyword. A simple if statement takes this form:

if (<expression>) {<statement>}


Here you are saying that if an expression resolves as true then do something. You can also use the else keyword to so something else if the expression being checked returns false. This "if... else..." form looks like this:

if (<expression>) {<statement>} else {<statement>}


In this case the expression will be evaluated, and if the (rounded) value is < = 0 ( it's false) the statement after else is executed, otherwise the initial statement is executed (it's true). It is a good habit to always put curly brackets around the statements in the if (otherwise only the first statement will be executed), and take a new line in the block for each statement, so the end code will have this form:

if (<expression>)
    {
    <statement>
    ...
    <statement>
    }
else
    {
    <statement>
    }


As a small example, consider the following code which will move the instance towards the position x=200 in the room:

if (x < 200)
    {
    x += 4;
    }
else
    {
    x -= 4;
    }


Note that you can also do compound checks in an if, ie: check various values or expressions in the same statement. These checks can use the various Combining Operators (&& and, || or, and ^^ xor). When you do this, GameMaker Studio 2 will evaluate each of them one at a time, and if depending on how they evaluate, then the rest may be skipped. For example:

if (keyboard_check_pressed(vk_enter) && instance_exists(obj_Player))
    {
    go = false
    alarm[0] = room_speed
    }


Here we are checking using the && "and" operator, so it's checking of both of the conditions in the if evaluate to true, and if the first one is false then the second one won't even be checked.


Conditional Operators


You can also perform conditional operations (also know as ternary operations), which are essentially a "shortcut" way of performing a basic if. It has the following syntax

variable = condition ? <expression1 (if true)> : <expression2 (if false)>


The conditional operator will return one of two given values depending on whether the condition evaluates to true or false, for example:

var temp_x = x < (room_width / 2) ? 32: (room_width - 32);


The above code will check the value of "x" against the value of half the room width and then if it is less it will set "temp_x" to 32 otherwise "temp_x" will be room width - 32. Here are a few more examples of use:

draw_text(x, y, "The fee is " + (global.Member ? "$2.00" : "$10.00"));
path_start((global.level > 10 ? path_hard : path_easy;), 2, path_action_reverse, true);
--hp <= 0 ? instance_destroy() : score += 10;


It is worth noting that you can nest conditional operations but that if you do then each operation will need to be enclosed in brackets, for example:

var c = a ? "foo" : (b ? "bar" : "wii"); // Correct
var c = a ? "foo" : b ? "bar" : "wii";   // Will cause an error
